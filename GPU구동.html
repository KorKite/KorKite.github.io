RTX30gpu 텐서플로우 구동을 위한 서버 완벽 설치 가이드

Edited By : 서버요정 고준서

이 모든 과정은 3일간의 텐서플로우 설치와 10번의 우분투 재설치를 통해 얻어진 방법입니다. 재설치하는거 진짜 힘드니까 GPU를 소중히 사용해주세요. 제발...

gpu는 쿠다 버전에 매우 예민하고, 텐서플로우는 더 예민합니다. 토치는 비교적 버전에 로버스트합니다. 서버를 통채로 초기화할 각오를 가지고 시작해주세요. 잘못되면 서버를 초기화하는게 제일 빠릅니다.

서버 설치 환경

  1. Ubuntu 18.04 (x86 아키텍쳐)
  1. GPU: RTX 3090
  1. 작동 확인 일자: 2021년 1월 18일
  1. 1080ti에서도 작동하는 것 확인(그대로 해도 됨)

서버를 처음부터 설치하는 경우 2번으로 가세요.

1. Uninstall driver

cudnn까지 설치된 상태면 조금 힘듭니다.

먼저 어떤 버전이 설치되어 있는지 확인해줍니다.

apt --installed list | grep nvidia-driver

# output
# nvidia-driver-450/bionic-updates,bionic-security,now 440.100-0ubuntu0.18.04.1 amd64 [installed]

output과 같은 형태로 nvidia-driver가 감지될 것입니다. 그러면 우리가 잘못 설치한 버전은 450입니다. 본인이 이 명령어를 통해 직접 빌드버전을 확인해주어야 합니다. 본인이 잘못 설치한 버전의 빌드번호를 확인해주세요. 이건 cuda의 세부 빌드 번호입니다. cuda11.1 뭐 이런거에 붙은 버전과는 다른 의미입니다.

sudo apt remove nvidia-driver-<설치된 버전>
sudo apt autoremove

만약 지우고싶은 빌드번호가 450이면 설치된 버전 자리에 450을 넣어주면 됩니다. 이 과정까지 끝나면 nvidia-driver가 지워집니다. nvidia-smi를 실행했을 때, nvidia명령어가 없다는 등의 메시지가 나오면 지워진 것입니다.

2. nvidia-driver installation

본격적으로 nvidia-driver를 설치해봅시다.

rtx30시리즈는 쿠다 11.1버전과 완벽하게 호환되도록 되어있습니다. cuda, cudnn, tensorflow, cracking까지의 과정이 완벽하게 통해야 설치와 함께 gpu를 사용할 수 있게 됩니다.

nvidia공식 사이트

https://developer.nvidia.com/cuda-11.1.1-download-archive

여기서 코드를 치셔도 됩니다. 저는 제가 설치했던 히스토리를 공유하겠습니다.

wget https://developer.download.nvidia.com/compute/cuda/11.1.1/local_installers/cuda-repo-ubuntu1804-11-1-local_11.1.1-455.32.00-1_amd64.deb
sudo dpkg -i cuda-repo-ubuntu1804-11-1-local_11.1.1-455.32.00-1_amd64.deb
sudo apt-key add /var/cuda-repo-ubuntu1804-11-1-local/7fa2af80.pub
sudo apt-get update
sudo apt-get -y install cuda

정확하게 버전을 확인해서 밑줄 그은 걸 다운로드 해주세요.

2. 사용했던 파일

이건 제가 다운로드 받은 파일인데, 정 안되실때만 시도하시고 이왕이면 서버에서 다운로드 하시기를 추천드립니다. (느리고 버전이 틀릴 수도 있어요, 래퍼를 쓰는 습관!)

다운로드 이후 서버로 업로드 합니다. 업로드 방법은 scp를 사용하거나 파일질라, cyberduck 등을 활용해주세요.

cudnn-11.1-linux-x64-v8.0.5.39.tgz

이 파일이 있는 디렉토리에서 아래 명령어를 실행해주세요.

tar -xzvf cudnn-11.1-linux-x64-v8.0.5.39.tgz #tgz압축 파일을 해제하는 코드
#필요한 파일을 cuda 경로로 복사
sudo cp cuda/include/cudnn*.h /usr/local/cuda/include
sudo cp cuda/lib64/libcudnn* /usr/local/cuda/lib64

#파일의 권한을 변경하여 서버에서 인식할 수 있도록 변경
sudo chmod a+r /usr/local/cuda/include/cudnn*.h /usr/local/cuda/lib64/libcudnn*

3. 쿠다 11.1에 맞는 텐서플로우 설치하기

텐서플로우가 예민해서 아무 버전이나 실행가능하지 않습니다. 그리고 텐서플로우는 globally하게 설치할 경우, 잘못 설치되었을 때, 완벽하게 지우기가 매우매우 어렵습니다. 절대로 globally하게 설치하지 않습니다. 절대로 입니다.

virtualenv를 사용하여 가상환경 하나를 열어줍니다. 가상환경 열줄 모르면 다음 래퍼를 확인하세요.

virtualenv사용하여 가상환경 열기

가상환경이 꼭 실행된 상태에서 하세요. 가상환경이 실행중인지 확인하세요. 가상환경에 설치하지 않고 전역으로 설치하다 실수 할 경우, 서버를 엎고 새로 설치해야합니다. 

pip3 install tf-nightly==2.5.0.dev20201116

이 코드를 활용해서 제가 안전성을 확인한 버전의 텐서플로우를 설치합니다. tf-nightly는 배포버전입니다. 

# 2021-05-29
# Jiwon Kang
# 2.5.0 버전이 정식으로 배포됐습니다. 해당 버전으로 설치시 아래 테스트 코드의 리턴값이 True가 나옵니다. 
pip3 install tensorflow_gpu==2.5.0

이렇게 설치를 하고 서버에 python3를 치면

이렇게 나옵니다.

이 상태로 아래를 한줄씩 쳐서 실행합니다.

import tensorflow as tf # 텐서플로우 불러오기
tf.test.is_gpu_available() # 텐서플로우가 gpu를 사용할 수 있는지 보여주는 코드

분명 설치를 다 했으나 아래의 코드를 치면 False가 나옵니다. 그 이유는 cudnn의 버전 중 일부가 텐서플로우가 찾기를 기대하는 버전보다 뒷 버전이어서 그럽니다. 이제 이 에러를 약간 크랙킹해보겠습니다.

4. 쿠다 크래킹을 통해 텐서플로우 인식시키기

False가 나오는 이유는 cudnn의 버전 중 하나가 7이어야 하는데 8로 설치가 되서 발생합니다. 아마 제가 시킨대로 설치하실 경우, 무조건 이게 뜹니다. False가 뜬 곳에 위에 에러 내역을 꼼꼼하게 읽어보시면 파일 하나가 인식이 안되었다는 것을 확인할 수 있습니다. (꼭 읽어보세요. 이거 읽고 크래킹한겁니다!)

아래 코드를 실행합니다.

sudo cp /usr/local/cuda-11.1/lib64/libcusolver.so.11 /usr/local/cuda-11.1/lib64/libcusolver.so.10

에러 코드를 보면 libcusolver.so.10 파일이 없다라고 나올겁니다. 이 파일이 없으면 가장 비슷한 파일을 이 이름으로 복사해서 인식되도록 만들면 되겠다고 생각해서 가장 비슷한 libcusolver.so.11을 libcusolver.so.10으로 복사해서 libcusolver.so.10파일을 만들어준 것입니다. 이를 통해 텐서플로우를 속일 수 있습니다.

다시 3번으로 돌아가 python3을 켜서 gpu를 사용할 수 있는지 보여주는 코드를 실행해보면 놀랍게도 true가 나오게 됩니다. 이 True를 확인했다면 축하합니다! 설치가 완료된 것입니다.
